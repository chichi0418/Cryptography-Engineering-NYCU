## Problem 1 — Stream Cipher Cryptanalysis (6 points)

### Background

A NTHU legacy industrial monitoring system uses a simple stream cipher. Due to an implementation error, the **same keystream S is reused** across different sessions. Three ciphertexts encrypted with the same keystream S are intercepted (all values in ASCII hex):

| Ciphertext | Bytes (hex) |
|------------|-------------|
| C1 | `03 16 0A 0C 0D 18 17 1F 0F 0D 11 05 03 00 0E 00` |
| C2 | `05 0D 10 08 0A 1A 01 18 0F 09 14 05 05 17 0F 00` |
| C3 | `0A 0A 0C 1C 09 09 00 0D 04 0B 11 18 1D 05 04 1D` |

All three ciphertexts are 16 bytes long.

---

### Task 1 — Theoretical Analysis (2 points)

Explain the **crib-dragging** attack technique:

- An attacker XORs two ciphertexts: `C1 ⊕ C2 = P1 ⊕ P2` (keystream cancels out).
- A guessed word (called a **Crib**, e.g., `"the"` or `"login"`) is "slid" (shifted position-by-position) over the result of `C1 ⊕ C2`.
- **Requirement:** Explain **why**, when the Crib is placed at the correct position, the resulting output at the other plaintext position exhibits **semantic readability** (i.e., produces recognizable English text).

---

### Task 2 — Practical Walkthrough (1 point)

**Known plaintext assumption:** P1 starts with `"BRIGHT SUN ARRIVES"` (uppercase ASCII).

**(a)** Use P1 and C1 to recover the **first 16 bytes of keystream S**:
- Formula: `S[i] = C1[i] ⊕ P1[i]`

**(b)** Use the recovered S to **decrypt C2 and C3**:
- Formula: `P2[i] = C2[i] ⊕ S[i]`, `P3[i] = C3[i] ⊕ S[i]`
- Assess whether the resulting plaintexts **make sense** (are readable English).

---

### Task 3 — Automated Cryptanalysis Program (3 points)

**File to submit:** `<student_id>_1_3.py`

Design an automated attack program that works **without any known plaintext**. The program must leverage one or both of:

1. **English letter frequency statistics** — e.g., score candidate plaintexts by how closely their character distribution matches natural English (high frequency of `'e'`, `'t'`, `'a'`, etc.).
2. **ASCII characteristics of the space character `' '` (0x20)** — spaces are the most common character in English text; XOR-ing a space with a letter flips the case (e.g., `'A' ⊕ ' ' = 'a'`), which can be used to detect likely keystream bytes.

**Expected behavior:**
- Input: the three ciphertexts C1, C2, C3.
- Output: the most likely keystream S and/or the decrypted plaintexts P1, P2, P3.
- The program should automate the crib-dragging or frequency-analysis process to identify the correct keystream bytes.

## Problem 2 — RSA Weak Key Recovery (4 points + 2 bonus)

### Background

Two IoT sensors use RSA keys generated by a **weak random number generator**. An attacker intercepts the following public keys:

| | Value |
|-|-------|
| **Modulus n1** | `0x61aa9a3bcb1e80b5a50ca09d8774ef0deba55e66e6bb90a835051256072ff701e51e09be8c339f4810c4abb9a4b1b22f` |
| **Public Exponent e1** | `65537` (0x10001) |
| **Modulus n2** | `0x72366ead2204584a49e1104941921c810a79bbfe84c0afeb2d873d38d3ca739c3a501936d006d9faaac5815219f94b51` |
| **Public Exponent e2** | `65537` (0x10001) |

Both moduli are 384-bit RSA keys.

---

### Task 1 — Identify the Flaw (1 point)

Identify the **likely flaw** in the RSA key generation process.

- Hint: When a weak RNG is used, different key pairs may share a **common prime factor** between their moduli. Computing `gcd(n1, n2)` directly yields that shared prime, breaking both keys instantly.

---

### Task 2 — Recover the Factorization (1 point)

Recover the prime factorization of the affected modulus/moduli:

- Compute `p = gcd(n1, n2)`
- Derive `q1 = n1 / p` and `q2 = n2 / p`
- Verify: `p × q1 == n1` and `p × q2 == n2`

---

### Task 3 — Compute Private Keys (1 point)

Compute the private exponents d1 and d2:

- `φ(n1) = (p − 1)(q1 − 1)`
- `φ(n2) = (p − 1)(q2 − 1)`
- `d1 = e1⁻¹ mod φ(n1)`
- `d2 = e2⁻¹ mod φ(n2)`

---

### Task 4 — Show All Steps (1 point)

Present all intermediate values and justify each step clearly in the PDF submission.

---

### Bonus — Real-World Case Study (2 points)

Find a **real-world published case** on the web where RSA private keys were recovered due to the same shared-prime vulnerability (e.g., from weak entropy in embedded devices/IoT). Demonstrate the key recovery process on that real example.
